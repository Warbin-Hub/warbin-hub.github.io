Software Engineering of UML

2022-10-17
notes-Curriculum Software Engineering

一、定义

UML是Unified Model Language的缩写，中文是统一建模语言，是由一整套图表组成的标准化建模语言。

二、类型

  • UML图分为结构图和行为图。
  • 结构图分为类图、轮廓图、组件图、组合结构图、对象图、部署图、包图。
  • 行为图又分活动图、用例图、状态机图和交互图。
  • 交互图又分为序列图、时序图、通讯图、交互概览图。

四、UML图

1. 类图

1.1 概念

类图是一切面向对象方法的核心建模工具。类图描述了系统中对象的类型以及它们之间存在的各种静态关系。

1.2 目的

用来表示类、接口以及它们之间的静态结构和关系。

1.3 工具介绍
1.4 关系介绍

2. 组件图

2.1 概念

描绘了系统中组件提供的、需要的接口、端口等，以及它们之间的关系。

2.2 目的

用来展示各个组件之间的依赖关系。

2.3 工具介绍

3. 对象图

3.1 概念

对象图是类图的一个实例，是系统在某个时间点的详细状态的快照。

3.2 目的

用来表示两个或者多个对象之间在某一时刻之间的关系。

3.3 工具介绍

4. 轮廓图

4.1 概念

轮廓图提供了一种通用的扩展机制，用于为特定域和平台定制UML模型。

4.2 目的

用于在特定领域中构建UML模型。

4.3 工具介绍

5. 组合结构图

5.1 概念

描述了一个”组合结构”的内部结构，以及他们之间的关系。这个”组合结构”可以是系统的一部分，或者一个整体。

5.2 目的

用来表示系统中逻辑上的”组合结构”。

5.3 工具介绍

6. 部署图

6.1 概念

描述了系统内部的软件如何分布在不同的节点上。

6.2 目的

用来表示软件和硬件的映射关系。

6.3 工具介绍

7. 包图

7.1 概念

描绘了系统在包层面上的结构设计。

7.2 目的

用来表示包和包之间的依赖关系。

7.3 工具介绍

8. 用例图

8.1 概念

用例图是指由参与者、用例，边界以及它们之间的关系构成的用于描述系统功能的视图。

8.2 目的

用来描述整个系统的功能。

8.3 工具介绍

9. 活动图

9.1 概念

描述了具体业务用例的实现流程。

9.2 目的

用来表示用例实现的工作流程。

9.3 工具介绍

10. 状态机图

10.1 概念

状态机图对一个单独对象的行为建模，指明对象在它的整个生命周期里，响应不同事件时，执行相关事件的顺序。

10.2 目的

用来表示指定对象，在整个生命周期，响应不同事件的不同状态。

10.3 工具介绍

11. 序列图

11.1 概念

序列图根据时间序列展示对象如何进行协作。它展示了在用例的特定场景中，对象如何与其他对象交互。

11.2 目的

通过描述对象之间发送消息的时间顺序显示多个对象之间的动态协作。

11.3 工具介绍

12. 时序图

12.1 概念

时序图被用来显示随时间变化，一个或多个元素的值或状态的更改。也显示时控事件之间的交互和管理它们的时间和期限约束。

12.2 目的

用来表示元素状态或者值随时间的变化而变化的视图。

12.3 工具介绍

13. 交互概览图

13.1 概念

交互概览图与活动图类似，但是它的节点是交互图。

13.2 目的

提供了控制流的概述。

13.3 工具介绍

14. 通讯图

14.1 概念

描述了收发消息的对象的组织关系，强调对象之间的合作关系而不是时间顺序。

14.2 目的

用来显示不同对象的关系。

14.3 工具介绍
